## Overview

Forex (FX) trading SDK for partners. Provides gRPC client/server interfaces for order management and currency pair queries. The Go code in `go/forexv1/` is **generated from Protocol Buffer definitions** — do not edit those files by hand.

## Architecture

### gRPC Services (package `forexv1`)

**OrderService** — forex order lifecycle:
- `SubmitOrder` — place an order (supports idempotency via `ref_id`)
- `CancelOrder` — cancel an existing order
- `GetOrderBookDepth` — market depth (bids/asks)
- `FilterClientOrders` — query orders with filters, pagination (limit/offset)
- `SubscribeOrderEvents` — server-streaming real-time order events

**PartnerService** — reference data:
- `GetCurrencyPairs` — available trading pairs and their specifications

### Package Layout

- `go/forexv1/` — generated protobuf + gRPC code (source protos: `forex/v1/order.proto`, `forex/v1/partner.proto`). Generated by `protoc-gen-go v1.36.11` / `protoc-gen-go-grpc v1.6.1`.
- `go/v1/` — intended for hand-written convenience client wrappers (currently a stub).

## Domain Conventions

- All financial values (quantities, rates, amounts) are **strings** for arbitrary decimal precision.
- Every operation is scoped to a `partner_id`.
- Client identification uses `partner_id` + `client_id` + `client_inn` (taxpayer ID).
- Market segments are int32: Retail, Corporate, Treasury.
- Order days use `YYYY-MM-DD` date strings.
